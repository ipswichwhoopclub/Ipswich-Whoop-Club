FIREBASE SETUP GUIDE — DO THIS BEFORE GOING LIVE
1
CREATE A FIREBASE PROJECT

Go to console.firebase.google.com, click Add project, name it ipswich-whoop-club and follow the prompts. You can disable Google Analytics if you don't need it.

2
ENABLE EMAIL/PASSWORD AUTHENTICATION

In your Firebase project, go to Authentication → Sign-in method and enable Email/Password. This is the provider used by the login form on this page. You'll add pilot accounts manually from the Authentication → Users tab, or build a registration flow later.

3
REGISTER THIS SITE AS A WEB APP & COPY YOUR CONFIG

Go to Project Settings → Your apps → Add app → Web. Name it IWC Portal. Firebase will give you a firebaseConfig object. Paste it into the firebaseConfig variable in the <script> block at the bottom of this file, replacing the placeholder values. Also add ipswichwhoop.club to Authentication → Settings → Authorized domains.

4
CREATE YOUR FIRESTORE DATABASE

Go to Firestore Database → Create database. Choose Start in production mode and pick the australia-southeast1 region. This will store pilot profiles, membership data and future records. You'll set security rules later when you build the member portal dashboard.

5
CREATE A PORTAL DASHBOARD PAGE

Create a new file called portal-dashboard.html. After a successful login, the script below will redirect to it. That page should call onAuthStateChanged on load — if there's no logged-in user, redirect back to portal.html. This is how you gate members-only content.
